## 1. Overview

E-Ijara (also written as E-Ejara) is a government-operated digital platform integrated with Uzbekistan's State Tax Service. It serves as the **sole mandatory channel** for registering real estate lease agreements with the tax authorities. The system ensures transparency, facilitates tax compliance, and provides legal validity to rental agreements.

### What E-Ijara Does

| Function                  | Description                                                   |
| ------------------------- | ------------------------------------------------------------- |
| **Lease Registration**    | Register new lease, sublease, and free-use agreements         |
| **Lease Management**      | Modify, extend, or terminate existing agreements              |
| **Tax Reporting**         | Automatically reports rental income data to tax authorities   |
| **Certificate Issuance**  | Generates an official Registration Certificate with QR code   |
| **Cadastral Integration** | Auto-fills property data from the national cadastral database |

### Who Must Use It

- **Individual property owners** (physical persons) renting out residential or commercial property
- **Individual entrepreneurs** leasing business premises
- **Legal entities** (companies) leasing real estate
- **Government property managers** renting out state-owned assets

## 2. Prerequisites

Before starting the registration process, ensure you have completed the following:

### 2.1. Obtain an Electronic Digital Signature (EDS)

- [ ] **Physical persons / Individual entrepreneurs** â†’ Obtain an EDS for a physical person
- [ ] **Legal entities** â†’ Obtain an EDS for a legal entity
- [ ] EDS can be obtained from authorized certification centers (e.g., via [id.egov.uz](https://id.egov.uz))

### 2.2. Register on the E-Government Identification System

- [ ] Create an account on [id.egov.uz](https://id.egov.uz) if you have not done so previously
- [ ] This account enables Single Sign-On (SSO) across government platforms

### 2.3. Ensure Cadastral Registration of the Property

- [ ] The property **must** be registered with the **State Cadastral Chamber**
- [ ] It must have a valid **cadastral number**
- [ ] If not registered, apply to the cadastral authorities **before** attempting to register a lease on E-Ijara
- [ ] For non-standard spaces (basements, attics, rooftops), ensure they have **separate cadastral registration**

### 2.4. Prepare Required Information

| Information Needed            | Details                                                                      |
| ----------------------------- | ---------------------------------------------------------------------------- |
| **Property cadastral number** | Unique ID from the cadastral registry                                        |
| **Lessee identification**     | INN (for legal entities) or PINFL (for individuals)                          |
| **Lease terms**               | Start date, end date, duration                                               |
| **Rental amount**             | Monthly fee in UZS or multiples of BRV (Base Calculation Value)              |
| **Purpose of use**            | Office, warehouse, production, public catering, services, residential, other |
| **Area being leased**         | Full property or partial (specify size in mÂ²)                                |

## 6. Tax Implications

### 6.1. For Individual Property Owners (Physical Persons)

| Parameter     | Details                                                     |
| ------------- | ----------------------------------------------------------- |
| **Tax type**  | Personal income tax on rental income                        |
| **Tax rate**  | 12% of rental income (standard rate for individuals)        |
| **Payment**   | Through the tax authority based on registered income        |
| **Reporting** | Automatically reported via E-Ijara to the State Tax Service |

### 6.2. For Individual Entrepreneurs & Legal Entities

| Parameter     | Details                                                    |
| ------------- | ---------------------------------------------------------- |
| **Tax type**  | Depends on the entity's tax regime (general or simplified) |
| **VAT**       | May apply depending on turnover thresholds                 |
| **Reporting** | Included in regular tax declarations                       |

### 6.3. Consequences of Non-Registration

> [!CAUTION]
> **Failure to register a lease agreement on E-Ijara may result in:**
>
> - Tax audits and inspections
> - Administrative fines
> - Back-taxes calculated on estimated rental income
> - The lease agreement being considered legally invalid

## 7. Common Mistakes & Troubleshooting

| #   | Common Mistake                          | How to Avoid                                                                         |
| --- | --------------------------------------- | ------------------------------------------------------------------------------------ |
| 1   | Property not in cadastral database      | Register the property with the State Cadastral Chamber before using E-Ijara          |
| 2   | Using the wrong type of EDS             | Physical persons must use an individual EDS, legal entities must use a corporate EDS |
| 3   | Lessee not confirming within 5 days     | Notify the lessee immediately and remind them of the 5-day deadline                  |
| 4   | Missing the 3-day registration deadline | Register the agreement within 3 days of signing the physical contract                |
| 5   | Incorrect cadastral number entered      | Double-check the cadastral number against official property documents                |
| 6   | Not registering changes to the lease    | Always register additional agreements when terms change                              |
| 7   | Forgetting to terminate expired leases  | Formally terminate agreements on the platform to avoid ongoing tax obligations       |
